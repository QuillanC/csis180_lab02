  1. Most HTML elements have a start tag and an end tag with text content in between. But, the reading presented several elements that only have one tag (the start tag). Which elements only have one tag and why is it not necessary for them to have end tags. Void or empty elements do not need a close tag. Some of these are img, input, hr, area, link, br, meta, base, col, embed, keygen, param, source, track, wbr
  2. We used a meta tag in lab to defined the character set. Why would HTML allow you to use a different character set? Hint: Think about the fact that HTML is for the World Wide Web. HTML allows the use of different character sets because not all languages use the same set of characters, and HTML is used world wide.
  3. A web page title typically appears in two places (in the head using the title tag and in the body using an h1 header). What is the difference between these two places and why is it useful to put a meaningful title in both places.The difference is that the title tag appears on the page tab and the h1 header appears on the top of the page. The title tag is useful if you are multitasking with multiple tabs open. The h1 title gives the title significance.
  4. It is a common convention to name HTML files with all lowercase letters. Why is this convention helpful/useful?HTML files should be lowercase because the web servers operating systems are commonly case sensitive. So using all lowercase letters will not confuse the system.
  5. There are certain symbols that cannot be used in a URL. Thus, they should not be used in the name of an HTML file. List five symbols that cannot be used. Symbols that cannot be used in a URL include: #, %, ?, &, and "
